### 성능 개선
- 페이지 이동마다 몇 초씩 느리던 원인 파악: Tailwind CDN 스크립트가 매 페이지 로드마다 브라우저에서 실시간으로 CSS를 새로 계산하고 있었음 (Tailwind 공식적으로도 프로덕션에 쓰지 말라고 명시된 방식). → 빌드 시점에 클래스를 미리 스캔해 정적 CSS 파일로 컴파일해두는 방식으로 전환 (`tailwind.config.js`, `tailwind.input.css` 추가, `npx tailwindcss`로 빌드).
- 로딩 중 빈 화면 대신 스피너 + 문구가 있는 로딩 화면 디자인 추가.
- **가장 큰 변경**: 모든 공개 페이지를 브라우저 JS가 나중에 `/api/data`로 데이터를 가져와 채우는 방식에서, **Thymeleaf가 서버에서 Redis 데이터를 미리 읽어 완성된 HTML을 내려주는 방식(SSR)** 으로 전면 전환.
  - 새 파일: `SiteContentService.java` (Redis 데이터 서버사이드 조회 + store.js와 동일한 기본값 포팅), `DefaultData.java`, `GlobalModelAttributes.java` (로고/메뉴/사이트설정을 모든 페이지에 전역 주입하는 `@ControllerAdvice`).
  - `fragments/nav.html`, `fragments/footer.html`을 Thymeleaf로 직접 렌더링 (기존엔 JS `Store.initNav()`가 매번 새로 그렸음).
  - 배너 회전/달력 월 이동/아코디언 등 남은 상호작용은 `th:inline="javascript"`로 데이터를 페이지에 미리 심어두고 그 위에서만 동작하도록 변경 (네트워크 왕복 없음).
  - 이제 필요 없어진 로딩 스피너/`Store.reveal()` 장치 제거.
  - 이 과정에서 실제 버그 2개 발견 및 수정: Spring이 Map에서 아예 없는 키에 접근하면 예외를 던지는 것(회의록/양식 페이지 500 에러), Thymeleaf 삼항식에서 else 분기에 `${}`를 안 씌우면 변수 값이 아니라 코드 자체가 문자로 출력되는 버그(푸터에 "content.footerTitle" 글자가 그대로 보이던 문제).

### 디자인/UX
- 메인 배너에 크로스페이드 + 서서히 확대되는(켄 번즈) 효과 추가, 새로고침 시 첫 로딩에도 페이드인 적용.
- 홍익대 엠블럼 이미지를 파비콘으로 적용.
- 메인 페이지 달력/공지사항 섹션을 화면 가로폭 꽉 차게 변경.
- 스크롤하면 콘텐츠가 페이드업되는 효과가 6개 페이지에만 있던 것을 25개 공개 페이지 전체에 동일하게 적용.
- 내용이 짧은 페이지에서 푸터가 화면 중간에 뜨던 문제 → 항상 화면 맨 아래에 붙도록 sticky footer 레이아웃 적용.

### 진행 방식
- 매번 로컬에서 실제 프로덕션 Redis(Upstash)에 연결해 전체 라우트를 curl로 검증하고, 서버 로그에서 에러 없는지 확인한 뒤에만 커밋·push. (`UPSTASH_REDIS_URL`은 대화 중 사용자가 다시 알려준 값을 사용 — 파일에는 저장하지 않음.)
- Render는 GitHub push 감지 후 자동 재배포 (Docker 빌드라 보통 3~8분 소요).
